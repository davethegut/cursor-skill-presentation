The Problem at Scale

As customers enable more detection rules, alert volume compounds. Analysts spend the majority of their triage time on repetitive manual work.

60-80%
of alerts in production environments describe incidents already represented by other alerts
Manual
Alert-to-case assignment requires analyst judgment for every alert, even obviously related ones
Full Reset
Attack Discovery regenerates from scratch every run — 500 analyzed + 10 new = re-analyze all 510

The core issue: Cases and Attack Discovery are powerful features that operate in isolation. The analyst is the glue — and that doesn't scale.

Based on SOC operational patterns observed across production-scale Elastic Security deployments. Source: Epic #16339.

Before & After

Today — Disconnected Features
  • Alert fires — analyst reviews individually
  • Analyst manually searches for a related case
  • Analyst attaches to case or creates a new one
  • Navigates to Attack Discovery as a separate step
  • AD regenerates all alerts from scratch each time
  • Results aren't linked back to the case
  • Repeat for every alert, every shift
After — Automated Pipeline
  • Alert fires — pipeline deduplicates upstream
  • Entities extracted (hosts, users, IPs, hashes)
  • Scored against open cases via entity matching
  • Attached to the right case automatically
  • Incremental AD processes only the new evidence
  • Results linked as case comments automatically
  • Analyst opens case with complete, current context

Built on Elastic Workflows. The pipeline is implemented as composable Workflow steps — dedup, entity extraction, case matching, and AD triggering are all Workflow primitives.

Attack Discovery 2.0

This pipeline is the backbone of AD 2.0 — evolving Attack Discovery from a point-in-time tool into a continuous, case-aware intelligence layer.

Today: AD 1.0

  • Point-in-time analysis
  • 100-alert ceiling
  • Full regeneration every run
  • No connection to Cases
  • Manual trigger only

AD 2.0: Pipeline-Powered

  • Continuous incremental analysis
  • 50,000+ alerts via batching
  • Delta-only processing
  • Case-scoped auto-triggered
  • Workflow-integrated

What This Enables

  • AD runs automatically when alerts attach to a case
  • Insights linked as case comments
  • Entity-based investigation queries
  • Composable Workflow steps for custom pipelines

Bringing It Together

The Problem
  • Alert volume compounds with every detection rule enabled
  • Most alerts describe incidents already being investigated
  • Cases and AD don't talk to each other
  • AD regenerates from scratch — costly and context-losing
  • Analysts are the manual glue — doesn't scale
The Solution
  • Automated pipeline deduplicates, extracts, matches, and groups alerts
  • Cases receive right alerts automatically via entity scoring
  • AD becomes incremental, case-scoped, and auto-triggered
  • Workflow steps are composable — reusable by Agent Builder and custom automations
  • Analysts open investigations with complete context
